\section{Модификация ИС ОПСАПК на основе автоматизированного выбора модели источника ошибок.} \label{sect3_2}
Рассмотрим ИС ОПСАПК, база источников ошибок в которой содержит только модели из класса скрытых полумарковских моделей. Расширим БУИМ из ИМ ЦПК (см. рис. \ref{Deundyak-fig1}) модулем подбора математической модели источника ошибок. Этот модуль получает на вход поток ошибок, зарегистрированный в реальном канале связи. Далее на основе алгоритма, предложенного в разделе 3.999 (вход, выход, ...), модуль выбирает из базы моделей источников ошибок, находящейся в БИВВ, ту модель, которая генерирует потоки ошибок, наиболее близкие к полученному.
Расширенный таким образом БУИМ будем обозначать БУИМ(А).

Выбранная модулем подбора математической модели источника ошибок модель подается на блок БУИМ. БУИМ делает запрос в БОР о результатах экспериментов по исследованию и подбору средств помехоустойчивой защиты для выбранной модели. Если такие эксперименты не проводились, то выбранная модель подается на вход блока БИВВ, который осуществляет генерацию потоков ошибок с помощью этой модели в рамках имитационных экспериментов (см. раздел 1.1.999).

Модель источника ошибок может выбираться как из всего множества базовых моделей, так и из некоторого его подмножества, выбранного на основе каких-либо дополнительных соображений.

Заметим, что в работе [найти работу про окно] предлагается метод определения числа различных состояний частного случая скрытой полумарковской модели по последовательности наблюдений. Используя идеи работы [работы про окно], представляется возможным сузить число исследуемых базовых моделей до моделей с определенным числом состояний. Однако, в настоящей работе это вопрос не исследовался.
