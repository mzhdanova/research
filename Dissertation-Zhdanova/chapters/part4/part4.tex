\subsection{Схема модифицированной ИС ОПСАПК.}
Рассмотрим информационную систему оценки применимости схем алгебраического помехоустойчивого кодировния (ИС ОПСАПК) из раздела 33333, база источников ошибок (БМИО) в которой содержит только модели из класса скрытых полумарковских моделей. 

... эта информационная система ... делает ... (см раздел 1...)...

Расширим функции этой информационной системы возможностью анализировать приходящую канальную последовательность ошибок и находить для нее такую модель источника ошибок из БМИО, которая генерирует последовательности ошибок, близкие к полученной последовательности.

Для решения этой задачи расширим блок управления имитационной моделью (БУИМ) из ИС ОПСАПК модулем подбора математической модели источника ошибок (МПМИО). Основой построения такого модуля является разработанный в разделе \ref{chapt3_Inverse_Problem} алгоритм. Задача МПМИО состоит в выборе из набора моделей источников ошибок модели, наилучшим образом соответствующей реальной канальной последовательности. Расширенный блок обозначим БУИМ(А).

Модель источника ошибок может выбираться как из всего множества базовых моделей, так и из некоторого его подмножества, выбранного на основе каких-либо дополнительных соображений. Естественной представляется возможность сузить число исследуемых базовых моделей до моделей с определенным числом состояний, однако, в настоящей работе этот вопрос не рассматривался.

На рисунке \ref{fig:chapt4_isopsapk2} представлена схема потоков данных между элементами модифицированной информационной системы. Рассмотрим подробнее работу этой системы с точки зрения решения поставленной задачи подбора модели.

\includegraphics[height=100mm]{/part4/isopsapk_mod.jpg}

На вход блока управления имитационной моделью (БУИМ) подается последовательность наблюдений $O_{1:T}$. БУИМ передает эту последовательность МПМИО, который, используя предварительную информацию о последовательности $O_{1:T}$, определяет критерий $K$ выбора подмножества моделей источника ошибок $\Lambda_K=\{\lambda_i\}$ из БМИО, способных сгенерировать такую последовательность. Получив набор моделей источников ошибок $\Lambda_K=\{\lambda_i\}$, МПМИО с помощью метода подбора из предыдущего раздела выбирает из него наиболее адекватную модель $\lambda$.

Выбранная модель $\lambda$ возвращается в БУИМ, который запрашивает у БОР результаты $R_\lambda$ интересующих пользователя экспериментов, проведенных с участием этой модели. Если искомые результаты $R_\lambda$ найдены, то необходимость в проведении имитационных экспериментов отпадает. Полученные результаты $R_\lambda$ возвращаются пользователю. Если же эксперименты для выбранной модели еще не проводились, то модель $\lambda$ подается блоком управления на вход генератора потоков ошибок, который производит генерацию потока ошибок моделью $\lambda$, и БУИМ запускает имитационные эксперименты. Результаты проведенных экспериментов подаются на вход БОР и сохраняются в базе данных результатов экспериментов, после чего выдаются пользователю.

НПС - мини-ИС

- НПС содержит программную реализацию модуля выбора модели и предлагается упрощенное обращение к нему

- исследователю моделей потоков ошибок НПС предоставляет возможность обратиться к модулю из главы 2 и сгенерировать последовательность ошибок моделью с заданными им параметрами для дальнейшего исследования сторонними средствами

- исследователю моделей потоков ошибок НПС предоставляет средства статистического анализа моделей источника ошибок на предмет соответствия заданным последовательностями, реализованные в модуле главы 3

- кто является пользователем НПС:

1) отладчик новой версии информационной системы (которая отличается от старой, по существу, только добавлением модуля подбора)

2) разработчик новой версии ИС ОПСАПК, отличающейся от старой новой базой моделей шума (новые модели источника ошибок), ...

3) разработчик новой версии ИС ОПСАПК, отличающейся от старой возможностью определять, достаточно ли хороша подобранная модель и строить новую (лучшую) на ее основе.

4) автор диссертации для имитации работы ИС ОПСАПК, отладки программных комплексов глав 2 и 3

5) потенциальный пользователь ИС, который хочет решать те задачи, которые умеет решать информационная система, может пользоваться в исследовательских целях только НПС.
